## 3. データベース設計

### 3.1 テーブル定義

#### employees テーブル

| カラム名 | 型 | NULL | 制約 | 説明 |
|---------|---|------|-----|-----|
| employee_id | TEXT | NOT NULL | PRIMARY KEY | 社員ID（例: A0001） |
| name | TEXT | NOT NULL | - | 氏名 |
| name_kana | TEXT | NOT NULL | - | 氏名カナ |
| department | TEXT | NOT NULL | - | 部署名 |
| position | TEXT | NOT NULL | - | 役職 |
| hire_date | TEXT | NOT NULL | - | 入社日（YYYY-MM-DD形式） |
| salary | INTEGER | NOT NULL | - | 給与（整数、円単位） |
| email | TEXT | NOT NULL | UNIQUE | メールアドレス |
| phone | TEXT | NULL | - | 電話番号（ハイフン含む） |
| postal_code | TEXT | NULL | - | 郵便番号（ハイフン含む） |
| address | TEXT | NULL | - | 住所 |
| notes | TEXT | NULL | - | 備考 |
| created_at | TEXT | NOT NULL | DEFAULT CURRENT_TIMESTAMP | 登録日時 |
| updated_at | TEXT | NOT NULL | DEFAULT CURRENT_TIMESTAMP | 更新日時 |

### 3.2 インデックス

```sql
CREATE INDEX idx_employee_name ON employees(name);
CREATE INDEX idx_employee_department ON employees(department);
CREATE INDEX idx_employee_hire_date ON employees(hire_date);
```

## 5. エラーハンドリング設計

### 5.1 エラー分類

| エラーレベル | 説明 | 対応方法 |
|------------|------|---------|
| CRITICAL | システム起動不可の致命的エラー | ログ出力 + プログラム停止 |
| ERROR | 処理失敗（継続は可能） | ログ出力 + ユーザーにエラー表示 |
| WARNING | 警告（処理は継続） | ログ出力 + 処理継続 |
| INFO | 通常動作の記録 | ログ出力のみ |

### 5.2 主なエラーケース

| エラーコード | エラー内容 | エラーレベル | 対応 |
|------------|----------|-----------|-----|
| E001 | データベース接続失敗 | CRITICAL | ログ出力 + 500エラー |
| E002 | テーブル作成失敗 | CRITICAL | ログ出力 + 500エラー |
| E003 | CSVファイル読み込み失敗 | ERROR | ログ出力 + エラー画面 |
| E004 | バリデーションエラー | WARNING | ログ出力 + フォーム再表示 |
| E005 | 社員ID重複 | ERROR | ログ出力 + エラーメッセージ表示 |
| E006 | メールアドレス重複 | ERROR | ログ出力 + エラーメッセージ表示 |
| E007 | 存在しない社員IDへのアクセス | ERROR | ログ出力 + 404エラー |

## 6. ログ設計

### 6.1 ログ出力先

```
logs/
  ├── app.log          # アプリケーションログ
  ├── error.log        # エラーログ
  └── access.log       # アクセスログ（オプション）
```

### 6.2 ログフォーマット

```
[%(asctime)s] %(levelname)s in%(module)s: %(message)s
```

例:
```
[2026-03-22 10:15:30] INFO in database_manager: Database initialized successfully
[2026-03-22 10:16:45] ERROR in csv_handler: CSV file not found: data/employees.csv
```

### 6.3 ログローテーション

- **最大ファイルサイズ**: 10MB
- **保持世代数**: 5世代
- **形式**: `app.log`, `app.log.1`, `app.log.2`, ...

## 8. CSV仕様

### 8.1 CSVファイル形式

#### ファイル名
`sample_employees.csv`（任意のファイル名可）

#### 文字コード
UTF-8（BOM無し）

#### ヘッダー行
必須（1行目にカラム名を記載）

#### データ例

```csv
社員ID,氏名,氏名カナ,部署,役職,入社日,給与,メールアドレス
A0001,山田太郎,ヤマダタロウ,営業部,部長,2010-04-01,8000000,yamada@example.com
A0002,佐藤花子,サトウハナコ,開発部,課長,2015-07-15,6500000,sato@example.com
```

#### バリデーションルール

| 項目 | ルール |
|-----|-------|
| 社員ID | 必須、英数字5文字、先頭はアルファベット |
| 氏名 | 必須、50文字以内 |
| 氏名カナ | 必須、全角カタカナ、100文字以内 |
| 部署 | 必須、100文字以内 |
| 役職 | 必須、50文字以内 |
| 入社日 | 必須、YYYY-MM-DD形式、1900-01-01～今日 |
| 給与 | 必須、0～999999999の整数 |
| メールアドレス | 必須、メール形式、255文字以内 |
